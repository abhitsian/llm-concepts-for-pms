Inference

Module 10 of From Sand to Software
┈┈┈┈

Training creates the model. Inference is using it. Every time you send a message to Claude, GPT-4, or any AI system, you run inference: the model processes your input through billions of parameters and generates a response, one token at a time. Training happens once and costs millions. Inference happens billions of times per day and never stops.

What Happens in 500 Milliseconds

Phase 1 — Prefill: The model reads the entire input (system prompt + conversation history + your message) and builds a compressed representation. This is compute-heavy and parallelizable. For 10,000 tokens of input, prefill takes 200–400ms. The model stores intermediate computations in the KV cache — its working memory for this conversation.

Phase 2 — Decode: The model generates the response, one token at a time. Each token requires a full forward pass through billions of parameters. This is sequential and memory-bandwidth-bound — the chip mostly waits for data from HBM. A 300-token response takes 1–2 seconds. This is why AI responses appear to "type" themselves out — the model is generating tokens as fast as memory bandwidth allows.

The KV cache grows with context length. Longer conversations = more memory consumed = fewer concurrent users per GPU. This is the physics behind rate limits and context window pricing tiers.

The Five Levers

Every serious AI lab has teams of 100+ engineers optimizing these five dimensions:

1. Batching — Process many queries simultaneously. A single query uses ~5% of GPU capacity. vLLM's continuous batching achieves 23x throughput improvement. Critical insight: low-traffic features are disproportionately expensive because batches don't fill. Launching AI to 100% of users can make unit economics better, not worse.

2. Quantization — Lower precision (FP8 instead of FP32) yields ~4x throughput with minimal quality loss. INT4 with custom kernels pushes further.

3. KV Cache Management — Cache repeated content (system prompts, common prefixes) instead of recomputing. If every user gets the same 2,000-token system prompt and you cache it, you eliminate the most expensive part of prefill. A 500-token system prompt vs. 5,000 tokens is ~10x cost difference on prefill. That's a design decision, not just an engineering one.

4. Speculative Decoding — Draft model predicts, main model verifies in parallel. 2–3x faster, zero quality loss.

5. Model Routing — Route easy queries to small cheap models, hard queries to frontier models. 80% of queries may not need the most powerful model. FAQ lookups don't need Claude Opus. Routing them to a cheaper model while reserving Opus for complex reasoning is a product architecture decision with direct margin impact.

The Cost-Capability-Speed Triangle

For every AI feature, you can optimize for two of three. Never all three simultaneously:

Capability (frontier model, full context) × Cost (token price, GPU hours) × Speed (time to first token, TPS).

A complex workflow agent: optimize capability + reliability, sacrifice speed (users expect to wait). Live search suggestions: optimize speed + cost, sacrifice capability (smaller model is fine). FAQ retrieval: optimize cost + speed, use a 7B model. Document analysis: optimize capability + speed, sacrifice cost (premium use case). This is a product decision per feature, not one model for everything.

Inference Is OpEx

Old SaaS: developer cost dominates COGS → 80%+ gross margins. Scale improves margins. AI SaaS: GPU cost dominates COGS → margins compress to 40–60%. The more successful the product, the higher the OpEx. Patel's mental model #4: "AI breaks SaaS economics."

Anthropic needs approximately 4–5 gigawatts just to serve projected revenue. At $10–13 billion per gigawatt per year in rental costs, inference economics determine whether AI labs survive.
