OpenAI's Billion-Dollar Dependencies

Deep Dive 3 — From Sand to Software
┈┈┈┈

OpenAI is the most famous AI company in the world. It's also one of the most dependent. Unlike Google (which builds its own chips and data centers) or Meta (which runs its own infrastructure at massive scale), OpenAI doesn't own the physical stack underneath its products. Every token ChatGPT generates runs on someone else's hardware, in someone else's data centers, on someone else's chips.

The Microsoft Marriage

Microsoft invested $13 billion in OpenAI between 2019 and 2023. In exchange, Microsoft got: exclusive cloud provider status (OpenAI runs on Azure), a license to integrate OpenAI models into Microsoft products (M365 Copilot, GitHub Copilot, Bing), and a revenue share on OpenAI's API business.

This was a brilliant deal for Microsoft. Azure became the default cloud for AI workloads overnight. GitHub Copilot got the best coding model in the world. Microsoft 365 Copilot — now at 15 million paid seats, growing 160% year-over-year — is powered by OpenAI's models. Azure AI contributes 16 percentage points to Azure's total growth.

For OpenAI, the deal provided survival capital and world-class infrastructure. But it created a structural dependency. OpenAI cannot leave Azure without renegotiating the most important partnership in AI. Azure's capacity constraints become OpenAI's capacity constraints. Azure's pricing floor becomes OpenAI's cost floor. When Microsoft says "a large share of AI capacity is allocated to our own products first," that's OpenAI's capacity being consumed by Microsoft Copilot before it reaches OpenAI's API customers.

The CoreWeave and Oracle Escape Routes

OpenAI has been diversifying its infrastructure away from sole dependence on Azure. CoreWeave — a neocloud valued at $35 billion despite being founded in 2017 — now provides significant compute for OpenAI's training runs. Oracle Cloud provides additional GPU capacity.

CoreWeave's value proposition: purpose-built GPU infrastructure with no legacy enterprise workloads competing for resources. They lease Nvidia GPUs in massive quantities and provide them as dedicated clusters. The risk: CoreWeave is highly leveraged, financed by debt secured against future GPU revenue. If AI demand softens, CoreWeave's financial model faces stress. And OpenAI's training runs face disruption.

The Stargate project — a reported $100+ billion infrastructure initiative — is OpenAI's attempt to eventually own its own compute destiny. But "eventually" is key. Building data centers takes 3–4 years. OpenAI needs compute now.

The Custom Chip Gambit

OpenAI is reportedly working with Broadcom to design custom AI chips — application-specific integrated circuits (ASICs) optimized for OpenAI's specific model architectures and inference patterns. The logic: if Google gets 30–40% cost advantage from custom TPUs, OpenAI could get a similar advantage from custom silicon.

But custom chip design takes 2–3 years from design start to production volume. TSMC fab capacity must be reserved. The chip design team must be hired and retained (semiconductor talent is scarce and expensive). And the software stack — the compiler, libraries, and optimization tools — must be built from scratch or adapted. Google has been doing this since 2013. OpenAI is starting now.

In the meantime, OpenAI runs on Nvidia GPUs — mostly H100s and B200s — rented from Azure, CoreWeave, and Oracle. Every Nvidia pricing decision, every TSMC allocation decision, and every power availability constraint directly affects OpenAI's cost structure and capacity.

The Business Model Tension

OpenAI's revenue reportedly crossed $12+ billion annualized in early 2026, growing rapidly. But so are costs. Training frontier models costs hundreds of millions to billions. Inference at ChatGPT's scale — hundreds of millions of users generating billions of tokens daily — requires enormous continuous compute.

The fundamental tension: OpenAI charges users $20–200/month for ChatGPT subscriptions and per-token API pricing. But it pays Nvidia, Microsoft, and CoreWeave for the GPU time those tokens consume. The margin between what users pay and what infrastructure costs is OpenAI's entire business. If Nvidia raises chip prices, OpenAI's margins compress. If Azure raises cloud prices, OpenAI's margins compress. OpenAI doesn't control its own cost structure.

Compare to Google: when Google improves TPU efficiency by 30%, 100% of that saving flows to Google's bottom line. When Nvidia improves GPU efficiency by 30%, the saving is split between Nvidia (higher chip ASP), the cloud provider (higher margin), and the end customer (lower API price). OpenAI captures only a fraction of each hardware improvement.

The Strategic Calculus

OpenAI's strategy is clear: use the best available infrastructure today (Nvidia GPUs via Azure/CoreWeave/Oracle), while investing in future independence (custom chips via Broadcom, owned data centers via Stargate). The question is whether the model layer commoditizes faster than OpenAI can build its own infrastructure layer.

If models commoditize before OpenAI achieves infrastructure independence, OpenAI becomes a high-cost provider of a commodity product — squeezed between Google (which runs models on custom chips) and Meta (which gives models away for free). If OpenAI maintains model leadership long enough to build its own stack, it becomes a vertically integrated AI company like Google.

The race is against time, commoditization, and the $600B/yr CapEx machine that its competitors already control.
